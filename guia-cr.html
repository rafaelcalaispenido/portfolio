Passo 1 de 5
O que é o Guia Operacional N1?
Uma ferramenta criada para apoiar o analista de suporte N1 na resolução dos chamados do eNotas, indicando o passo a passo correto com base no tipo de atendimento.
  • Você informa o número do ticket e os dados do atendimento.
  • A ferramenta identifica o fluxo correto e exibe o passo a passo que o analista deve seguir.
  • Ao final, lembra de aplicar a macro no Zendesk e oferece acesso ao Macro Mind para encontrar a macro certa.
  • Todas as consultas ficam registradas em Últimas Consultas e podem ser exportadas para Excel.

Use as setas abaixo para navegar entre os tópicos deste manual.

Passo 2 de 5
O que precisa ser preenchido?
Na aba Guia, preencha os campos abaixo em ordem. Os campos seguintes só aparecem depois que a Categoria do Atendimento é selecionada — e o passo a passo só é exibido quando todos os campos obrigatórios estão preenchidos.
Número do Ticket obrigatório
Ex: 1234567Somente números
  • Informe o número do ticket do Zendesk. Aceita somente números.
Categoria do Atendimento obrigatórioCampo 1
Ex: Configuração eNotas
  • É o primeiro campo a ser preenchido. Enquanto ele não for selecionado, os campos de busca e de Agent Contact Reason ficam ocultos.
Buscar fluxo por texto Atalho
Ex: "reembolso", "nota fiscal"...
  • Aparece logo abaixo da Categoria do Atendimento. Digite uma palavra-chave do assunto para encontrar o fluxo mais rápido, sem precisar navegar pelos menus de Agent Contact Reason.
  • A busca já considera apenas os fluxos da Categoria do Atendimento selecionada.
  • Ao clicar em um resultado, os campos eNotas | Agent Contact Reason (e Subcategoria, se houver) são preenchidos automaticamente — a Categoria do Atendimento permanece a que você já escolheu.
eNotas | Agent Contact Reason obrigatórioCampo 2
Ex: Notas Fiscais
Agent Contact Reason — Subcategoria Campo 3
Selecione a subcategoria após escolher o Agent CR
  • eNotas | Agent Contact Reason é a categoria principal do contato (ex: Notas Fiscais, Configurações). Pode ser preenchido manualmente ou através da busca por texto.
  • Subcategoria aparece automaticamente após selecionar o Agent Contact Reason e refina o tema do atendimento.
  • Se o Agent Contact Reason escolhido não tiver subcategorias cadastradas, o campo 3 fica desabilitado (cinza) com a mensagem "Não se aplica a este Agent CR" — não é necessário preencher nesse caso.
  • A combinação de Categoria + Agent Contact Reason (+ Subcategoria, quando houver) determina qual fluxo será exibido.

Se não houver fluxo cadastrado para a combinação selecionada, o guia exibirá uma mensagem informando que o caminho ainda não foi mapeado. Nesse caso, o responsável pelo Admin será notificado para inclusão do fluxo.

Incluir no contexto
Consultar Macro Mind Consultar Baymax
  • O passo com ícone ℹ (navy) é o contexto de uso — serve para o analista entender o cenário antes de iniciar. Não é um passo numerado.
  • Os passos 1, 2, 3... são as ações que devem ser executadas conforme o cenário do atendimento.
  • Cada passo é exibido como um card retrátil (accordion): clique no título para abrir/fechar a descrição. Use os botões "Expandir todos" / "Recolher todos" no topo da lista para abrir ou fechar tudo de uma vez.

Use o botão "Copiar guia" para copiar o passo a passo formatado e colar no ticket do Zendesk como ação interna.

Passo 4 de 5
Macro Mind e Baymax
No painel direito do resultado você encontra dois assistentes de IA para apoiar o atendimento — cada um pode ser consultado de forma independente.
Macro Mind
IA especializada em sugestão de macros do Zendesk. Clique em "Consultar Macro Mind" para abrir o bot já com o contexto do atendimento pré-preenchido.
Baymax
Assistente geral da Atlassian. Clique em "Consultar Baymax" para abrir o bot com o mesmo contexto enviado. Útil para dúvidas mais amplas sobre o processo ou produto.
Seleção de passos para o contexto
  • O passo "Contexto de uso" e os dados de Agent CR são sempre enviados automaticamente.
  • Os passos numerados aparecem com checkbox — todos marcados por padrão.
  • Desmarque os passos que não quer incluir no contexto antes de clicar no bot.
  • Isso permite refinar a consulta enviando apenas o que é relevante para aquele atendimento específico.
  • O Macro Mind finaliza a consulta perguntando qual macro usar — ele conhece a base de macros do Zendesk.
  • O Baymax finaliza perguntando quais passos/ações tomar com o cliente — sua base de conhecimento são os artigos do Help Center do eNotas, não macros.

Você pode consultar o Macro Mind e o Baymax de forma independente — não é necessário usar os dois ao mesmo tempo. Escolha o mais adequado para cada situação.

Passo 5 de 5
Busca por texto e atalhos para validar passos
Recursos pensados para deixar o uso mais rápido, tanto em atendimentos síncronos (chat/ligação) quanto assíncronos (ticket).
Buscar fluxo por texto
  • Campo exibido logo abaixo da Categoria do Atendimento, assim que ela é selecionada.
  • Digite uma palavra-chave do assunto (ex: "reembolso", "nota fiscal") para listar os fluxos correspondentes, sem precisar conhecer os nomes exatos de Agent Contact Reason / Subcategoria.
  • A lista de resultados já é filtrada pela Categoria do Atendimento selecionada.
  • Ao clicar em um resultado, os campos de Agent Contact Reason (e Subcategoria, se houver) são preenchidos automaticamente — a Categoria do Atendimento não é alterada.
"Validar a partir de" — selecionar passos em lote
  • No painel do bot (lado direito do resultado), há um seletor "Validar a partir de".
  • Ele permite marcar de uma vez todos os checkboxes a partir de um determinado passo, em vez de marcar/desmarcar um por um.
  • Útil quando o atendimento já avançou e alguns dos primeiros passos já foram cumpridos antes do analista abrir o guia — basta escolher a partir de qual passo deseja validar.

Esses atalhos foram criados para reduzir cliques tanto em atendimentos síncronos (chat/ligação, onde a velocidade é essencial) quanto em atendimentos assíncronos (ticket), tornando o uso da ferramenta mais natural no dia a dia do analista.
